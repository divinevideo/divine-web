# i18n/L10n Kickoff Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add client-side locale infrastructure, browser auto-detect plus persisted override, a language switcher, translation catalogs for 15 supported languages, and an initial translation pass over the shared shell and repeated core UI states.

**Architecture:** Use `i18next` and `react-i18next` with a small `src/lib/i18n/` layer that owns supported locale metadata, browser matching, persistence, and i18next initialization. Wire the provider into the app root, update `document.documentElement.lang` and `dir` from locale state, and convert shared shell/core state copy to semantic translation keys in a `common` namespace. Keep locale out of routes and leave long-form legal/editorial pages in English for now.

**Tech Stack:** TypeScript, React 18, React Router, i18next, react-i18next, Vitest, Testing Library
